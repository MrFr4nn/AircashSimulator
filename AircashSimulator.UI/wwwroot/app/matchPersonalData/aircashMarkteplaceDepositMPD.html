Aircash marketplace deposit - match personal data implementation

There are two possible approaches for the personal data match for the Aircash marketplace deposit flow:

  1. Aircash will do the data match.
  2. Partner will do the data match.

Aircash does the data match

Once we receive a response that the user exists within the partner system, we are then checking if the data for the user’s first name, last name and date of birth matches with the ones within our system. If there is a positive match, CreateAndConfirm method is automatically called. Otherwise, CreateAndConfirm method will not be called and the user will get a proper error message within his app.

Partner does the data match

If the user exists within partner system then they instantly need to call method CompareIdentity (Aircash Simulator) next.
After the partner gets confirmation from Aircash system that there is a match for CompareIdentity method, partner needs to respond to us with response for CheckPlayer.
In case that there is no match, then partner returns the response for CheckPlayer with agreed error code.
In this case, CreateAndConfirm method will not be called and the user will get a proper error message within his app.

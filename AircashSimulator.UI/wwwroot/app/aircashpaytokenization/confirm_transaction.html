ConfirmTransaction


Once the partner system performs authorization, the partner system must perform confirmation to complete the transaction within the timeframe. Confirmed transactions will be part of the settlement process with the partner. Confirmed transactions cannot be canceled but only refunded to the end-user.

URL: /api/AircashPay/ConfirmTransaction

Parameter Type Description Mandatory
PartnerID string PartnerID YES
PartnerTransactionID string ID that partner sends in the Authorization method. YES
Amount decimal Partial completion is possible in which case the partner should send the amount that the partner would like to confirm. Partner is not able to confirm transactions above authorization amount (in such case partner must create new authorization request and new confirmation). YES
CurrencyID int Currency ID (e.g. 191, 978) - must be the same as in authorization request YES
Signature string Signature by Aircash system confirming that Aircash is sending a request to the partner system YES

Aircash system will respond 200 OK in case of successful confirmation.

Parameter Type Description Mandatory
PartnerID string PartnerID YES
PartnerTransactionID string Same as requested ID YES
Amount decimal Confirmed amount, same as in the request. YES
CurrencyID int Currency ID (e.g. 191, 978) - must be the same as in authorization request YES
AircashTransactionID Guid Aircash transaction ID - will be the same as in authorization response. YES
Signature string Signature by Aircash system confirming that Aircash is sending a request to the partner system YES

Error code Error message
0 Invalid request
1 Unable to find the transaction
2 Confirm amount is higher than authorization amount
3 Transaction already confirmed
4 The transaction has been canceled and cannot be confirmed

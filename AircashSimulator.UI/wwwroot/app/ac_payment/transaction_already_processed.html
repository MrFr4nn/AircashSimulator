Transaction Already Processed

Option A

When receiving a CreateAndConfirmPayment request with the same TransactionID (this is Aircash transaction id) partner can return the same successful response.

Response example:

Response HTTP 200 OK
Option B

When receiving a CreateAndConfirmPayment request with the same TransactionID (this is Aircash transaction id) partner can return the error code for the already processed transaction.
Partner needs to provide Aircash with correct error code for the transaction that is already processed. After that transaction will be mark as a succesfull/finished.

Response example:

Response HTTP 200 OK

Payment process


This chapter describes the top up process using the Aircash application. The described process explains how the user can pay the invoice in the partner's web store using Aircash aplication

The transfer process consists of three steps:

  1. Within the Aircash application, the User selects services within the partner's web store
    • The partner should provide a link to the web store that will only have the Aircash payment method enabled (optionally the Aircash application can place the User-Agent in the header of all requests at a fixed value by which the merchant's website would know it opens within the Aircash application)
    • The Aircash system will open the specified link for the web store within the Aircash application
    • The user selects a list of services he wants to buy within the web store
    • User selects payment via Aircash payment method (only payment method)
  2. Payment for services
    • The web store, upon the user's request for payment, contacts the Aircash system and creates a request for payment
    • The Aircash system will return to the web store the URL to which the web store should direct the user
    • The Aircash mobile application will, upon directing the ser to the specified URL, require the authorization of the transaction within the Aircash application
  3. Confirmation of the transaction to the merchant
    • Upon successful authorization, the Aircash system will notify the partner system of the successfully completed transaction to the defined address

Methods called by partner system on the Aircash system

The partner should call the method to create a transaction on the Aircash system. Additionally, the partner can call the CheckTransactionStatus method to check the transaction. The CheckTransactionStatus method is optional if ConfirmTransaction is used where the Aircash system will send a confirmation to the partner system of successful payment


Methods called by Aircash on the partner system

The partner needs to open one REST method that receives JSON / XML objects in the request in order to successfully make a payment to the partner's account. This method is optional and the partner can use the CheckTransactionStatus method to check the transaction status

Methods called by partner system on the Aircash system

ConfirmTransaction

Once the user confirms the transaction Aircash system will send a confirmation message towards the partner system. Partner system should process that message and return a successful response in case of successful processing. In case Aircash system doesn’t receive successful response Aircash system will retry this request several times after which it will notify Aircash support upon unsuccessful confirmation.

Partner should provide an url where Aircash will send all transaction confirmations.

Test POST Defined by partner's system
Production POST Defined by partner's system

Aircash system expects an answer http answer 200 OK. In case the Aircash system gets any other response the Aircash system will call this method again. The method is called 10 times in a period of 6 hours after which the call is canceled and Aircash support is notified of the failed transaction confirmation.


CheckTransactionStatus

Optional method for checking the status of the transaction. This method has restrictions in the number of calls and in case the partner overuses this method, the Aircash system may reject requests. If a partner feels that they will call this method frequently, they should consult Aircash to see if the Aircash system will block it in requests.

Partner should provide an url where Aircash will send all transaction confirmations.

Production process


The production process consists of several steps that take place sequentially:

  1. All acceptance criteria are checked and are performing in line with expectations
  2. Aircash will enable partner in production
  3. If possible partner will generate one test transaction in production
  4. If all tests are successfully completed, Aircash and Partner will release the service to all their customers

Acceptance criteria


To release the service in the production, you need to make several tests to see if the integration is well-made and ready for production

Scenario Test description Mandatory
Generating a transaction The partner system needs to successfully generate a transaction YES
Route users to the appropriate URL Upon successful generation of the transaction, the partner system should direct the user to the appropriate URL YES
Transaction confirmation The partner system must successfully enter the transaction into its system, upon sending the Aircash confirmation YES, if this method is implemented
Check the status of the transaction The partner system checks the status of the transaction and, after successful confirmation, enters the transaction into its system YES, if this method is implemented
